What is your favorite WordPress function? Why?

My favorite WordPress function of late is add_filter().

With it, a skilled developer can change nearly everything about WordPress’s built-in functions. A few examples:

  • WordPress galleries offer an easy way for users to assemble a group of media. However, galleries’ default markup can be difficult to work with. I like to use add_filter() to transform the markup into something that’s suitable for a slideshow.
  • WordPress extraneously wraps elements in paragraph tags, even images and videos. This isn’t very semantic. I like to use add_filter() to hook a function onto the the_content and parse out the extra <p> tags.
  • wp_title()’s default behavior doesn’t include the site’s title on a few views, often leaving pages with no title. This is an easy fix with add_filter(): a custom function can act on the wp_title hook to include the site’s title on every page, remedying the problem.

Name three good reasons to start a branch in a version control system.

  1. Working on a project’s next major version. Got some sweet modals and GIFs lined up for your site’s next big iteration? Better start a new branch. This way, your new features are isolated from the production branch. And, you can keep those features all to yourself until you’re ready to launch.
  2. Debugging something right now. What if – while fixing this bug – you inadvertently cause another new problem? Instead of working directly on the production branch, it’s safer to isolate your code in a new branch. Fix the bug and merge ‘em up.
  3. Trying out an experimental feature. Say you want to drop some new hotness on your project, but you’re not sure it will work. Start a new branch to play with that fire. Doesn’t work out? Toss your branch.

Why is it better to use CSS (rather than JavaScript, if possible) for a basic web page animation?

CSS animations and transitions run faster than their JavaScript counterparts.

A browser is able to execute CSS changes through the device’s GPU when relying on CSS3’s animation or transition. This is way faster than relying on JavaScript execution, which is interpreted through the browser, and therefore slower.

The difference is usually negligible on a desktop computer with a modern browser; however, the issue becomes obvious on mobile devices, where JavaScript execution is an order of magnitude slower than desktops. JavaScript animations on a phone are typically choppy, slow and hold up the scrolling.

Look at our portfolio and find something we've built that could be improved. Suggest how!

Equality House’s site looks great, reads great, and functions great; yet, there is a key area for improvement.

Let’s talk about performance, specifically stylesheets. Remember Yahoo!’s guidelines: minify files, use less HTTP requests, compress files, etc.

On the home page, the site loads five stylesheets. (There’s another one hiding on the product page.) One results from Typekit loading, so it can’t be eliminated. The other four, however, could be compressed into a single stylesheet. This would decrease file size (by minimizing plugins’ CSS) and reduce HTTP requests, making the page load and render faster. Here’s how to do this:

  1. Use WordPress’s wp_dequeue_style in the theme’s functions.php to remove plugins’ stylesheets.
  2. Import all plugins’ stylesheets into app.css and minimize it.

This is more work on the developer: when a vendor updates his or her plugin, you have to manually rebuild app.css. In the end, this would pay off in performance. (One could add an action function that fires every time something is updated. Plugin idea, anyone?)

There’s also a few server-side issues. None of the stylesheets have a Cache-Control header, nor are they gzip-ed. This means the browser potentially downloads stylesheets on every page load, and the stylesheets aren’t compressed by the server. WordPress is the likely culprit: it enjoys setting query strings on assets. Setting app.css’s queue function with the $ver parameter to NULL will drop the query string. If these problems persist, some tweaking of the server’s configuration file should do the trick.
