Name three good reasons to start a branch in a version control system.

  1. Branches keep your sandbox work out of your stable codebase.
  2. With branches, participants produce a cleaner more logical workflow.
  3. It is far easy to merge distributed code from multiple developers using branches.
  4. Hard drives die and a DVCS is an excellent backup.
  5. People leave companies and version control allows those companies to keep the code.

Full Disclosure: I have taken the lazy path before and started work on a small feature within the stable branch of an application. At just about the point of no return, I found myself scrambling to put fix a bug on that same branch. I had to copy directories, roll back code, fix the bug and try to piece it all together after. The lesson was learned.

Why is it better to use CSS (rather than JavaScript, if possible) for a basic web page animation?

CSS is almost always better supported on browsers (and more difficult to disable). CSS loads faster than Javascript, which is why all the cool kids load their JS at the end of their HTML. Many Javascript animations also require jQuery and some assortment of plugins which can be difficult to maintain. For most modern browsers I would imagine the difference to be negligible, but CSS animation requires a browser that can understand properties like @keyframe or animation. Sorry IE9.
